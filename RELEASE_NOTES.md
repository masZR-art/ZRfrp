# ZRfrp v2.2.6

- Desktop 将控制平台账号会话提升为应用级状态，兼容无节点登录与旧版按节点保存授权的数据，并让所有托管节点始终使用同一份最新令牌。
- 有效 Access Token 不再因“本次运行尚未续期”而被强制刷新；仅在临近过期或服务端实际返回 HTTP 401 时续期，修复界面显示已登录却提示授权失效的问题。
- 登录和续期成功后先持久化会话再同步节点；同步暂时失败不会丢失新令牌，失效会话会准确切换为未登录并打开登录窗口。
- 控制平台返回空响应、纯文本或 HTML 错误时不再触发 JSON 解析异常，HTTP 401 会稳定进入认证恢复流程。
- 服务端串行化登录会话创建与 Refresh Token 轮换，避免并发续期让刚签发的令牌立即失效。
- 验证码邮件模板在桌面和移动端均使用严格居中的 600px 响应式布局，缩小左右留白并收紧正文行距；旧默认模板启动后自动迁移。
- 验证码邮件显式使用 UTF-8 主题与正文编码，提升中文邮件客户端兼容性。

## v2.2.5

- Desktop 在服务端端口分配返回 HTTP 401 时自动刷新控制平台授权并重试，修复登录令牌轮换后无法启动连接的问题。
- 删除托管隧道释放端口租约遇到 HTTP 401 时采用相同的自动续期流程，避免旧 Access Token 导致删除失败。
- 控制平台 API 异常保留 HTTP 状态码；Refresh Token 同样失效时明确提示重新登录，且不会无限重试。

## v2.2.4

- 检测到包含旧错误占位符的模板时自动迁移为白底紫色标题栏的标准验证码邮件。
- 默认邮件改用表格布局和内联样式，提高 Gmail、QQ 邮箱及移动端邮件客户端兼容性。

## 功能

- 可视化管理多个 frp 节点。
- 创建、编辑、启用、停用、删除 frp 隧道。
- 支持 `tcp`、`udp`、`http`、`https` 隧道配置。
- 自动生成 `frpc.toml`，并在启动前调用 `frpc verify` 校验配置。
- 启动、停止并监控 `frpc.exe` 进程。
- 彩色运行日志，隧道启动成功后显示可连接地址并支持点击复制。
- 节点 TCP 延迟测速，支持启动时自动测速和手动一键测速。
- 托盘常驻，支持后台运行、恢复主界面、彻底退出和通道开关。
- 暗色无边框界面，支持浮动设置/编辑窗口。
- 自动检测、选择或下载安装本机 `frpc.exe`。
- 软件联网代理设置：系统代理、不使用代理、手动代理。
